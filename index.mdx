Minns Memory Layer is a **graph-native memory engine** for AI applications. Send messages or conversations, and Minns automatically extracts structured knowledge — locations, relationships, preferences, financial transactions — into a queryable graph.

## How it works

<Steps>
  <Step title="Send messages">
    Send messages as they arrive with `sendMessage()`, or ingest historical conversations in bulk with `ingestConversations()`.
  </Step>
  <Step title="Automatic knowledge extraction">
    Minns extracts structured facts from your messages — locations, relationships, preferences, financial transactions — and writes them as graph edges. No manual schema needed.
  </Step>
  <Step title="Query in natural language">
    Ask questions about your graph in plain English with `query()`. The pipeline classifies intent, resolves entities, and returns human-readable answers.
  </Step>
  <Step title="Knowledge grows over time">
    As more data flows in, the graph builds richer connections. Entity resolution ensures consistent identity across messages and sessions.
  </Step>
</Steps>

## Three core endpoints

| Endpoint | Use case | Description |
| :--- | :--- | :--- |
| `sendMessage()` | Real-time | Send individual messages as they arrive. Deferred compaction. |
| `ingestConversations()` | Bulk / historical | Ingest multiple sessions at once. Inline compaction. |
| `query()` | Retrieval | Ask questions in natural language. Get human-readable answers. |
